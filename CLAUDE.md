# 编码规范与人机交互原则
- 不要破坏现有功能.
- 一个函数不超过40行, 简洁优雅, Unix风格.
- 渐进式改动, 每次只改一个函数, 每次改动前先规划好让我确认
- main.go中, main函数一定在最下面
- 局部变量不要用大写, 倾向于使用极短的变量名（如 i, j, p, q）, 最好不超过6个字, snake命名法
- 全局变量用全大些, snake命名法
- 函数参数超过2个, 考虑用面向对象
- 所有成员函数和成员变量都要注释, 不要过短, 说明清楚. 注释开头不要重复函数名或变量名.
- git commit, 用中文说明
- type struct, 类名默认以大写开头
